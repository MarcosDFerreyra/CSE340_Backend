/**

 * A connection pool maintains a set of reusable database connections

const pool = new Pool({

/**
 * Common SSL Issue:

/**

 * export a reference to the pool object. This allows us to use the same name for the
 * export regardless of whether we are in development or production mode.

let db = null;

if (process.env.NODE_ENV === 'development' && process.env.ENABLE_SQL_LOGGING === 'true') {
    /**
     * In development mode, we wrap the pool to provide query logging.
     * This helps with debugging by showing all executed queries in the console.
     * 
     * The wrapper also adds timing information to help identify slow queries

        async close() {

        }

AI: You're on the right track with the general idea — let's sharpen it up a bit.

<%= %> escapes HTML — meaning if the value contains characters like <, >, or &, it converts them to safe entities so they display as literal text instead of being interpreted as HTML. That's why you use it for things like title, project.title, and the formatted date — you're just displaying plain text/data, and you don't want any risk of it being treated as actual HTML tags.

<%- %> does not escape — it outputs raw HTML as-is. That's why you use it for include('partials/header') — you actually want that included content to be interpreted as real HTML (tags, structure, etc.), not shown as literal text on the page.

Here's a question to check this really clicked: if project.title somehow contained something like <script>alert('hi')</script>, what would happen differently if you used <%= %> versus <%- %> to display it?

USER: It would take everything as text, it would show everything just as you see it there, it wouldn't show "hi".

AI: Exactly right — with <%= %>, it would just display the literal text <script>alert('hi')</script> on the page, harmless. But with <%- %>, that script tag would actually be interpreted as real HTML and the JavaScript would execute — that's a security risk called XSS (Cross-Site Scripting). So <%= %> isn't just about formatting — it's also protecting you from malicious data being injected and run in the browser.
